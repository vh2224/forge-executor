## Why this matters

Two transitive advisories affect reachable runtime code:

1. **undici WebSocket DoS (GHSA-vxpw-j846-p89q)** â€” `discord.js@14.26.4` (a direct
   dependency the daemon uses to open a live WebSocket to Discord) pulls
   `undici@6.24.1` (`<6.27.0`), vulnerable to a fragment-count DoS. The daemon's
   Discord bot maintains exactly this kind of long-lived WS connection. (Note: the
   repo's **root** `undici@^7.26.0` is already safe â€” this is specifically the
   copy bundled under discord.js.)
2. **protobufjs prototype pollution (GHSA-f38q-mgvj-vph7)** â€” `@google/genai`
   pulls `protobufjs@7.6.1` (`<=7.6.2`). Practical risk is low (the CVE is not
   known to allow code execution by itself), but the fix is a one-line override.

Both are closed with pnpm `overrides` in `pnpm-workspace.yaml` (this repo already
uses that block for security pins â€” Dependabot writes there, not to
`package.json`).

## Steps

### Step 1: Record the baseline

Run `pnpm audit` and confirm both advisories are present (undici `<6.27.0` via
discord.js; protobufjs `<=7.6.2` via @google/genai). Save the "before" output for

**Verify**: `pnpm audit` lists both.

### Step 2: Add scoped overrides

Add two entries to the `overrides` block. Use a **scoped** override for undici so
only the discord.js subtree is affected (the root undici 7.x stays as-is):

```yaml
overrides:
  uuid: ^11.1.1
  postcss: ^8.5.10
  "@mistralai/mistralai": 2.2.1
  gaxios: 7.1.4
  "c8>yargs": "^18.0.0"
  "discord.js>undici": ">=6.27.0"
  protobufjs: ">=7.6.3"
```

`protobufjs` can be a plain (unscoped) override â€” the only consumer is
@google/genai and >=7.6.3 is API-compatible.

If pnpm rejects `"discord.js>undici"` because undici is nested deeper than a
direct child of discord.js (it may live under `@discordjs/ws` / `@discordjs/rest`),
fall back to the deep-scoped form pnpm supports, e.g. `"discord.js>@discordjs/ws>undici"`,
or an unscoped `undici@6: ">=6.27.0"` selector. Use `pnpm why undici` (step 3) to
see the actual parent chain and pick the selector that pins **only** the 6.x copy,
never the root 7.x.

**Verify**: `pnpm install` â†’ exit 0.
